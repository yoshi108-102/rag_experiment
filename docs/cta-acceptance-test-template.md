# CTAインタビューエージェント 受入テストケース雛形

## 1. 使い方
- この雛形は、要件IDごとに受入テストを定義するためのテンプレート。
- まず「受入テスト計画」を埋め、次に「ケース一覧」を実案件向けに具体化する。
- 実施証跡（ログ、スクリーンショット、計測結果）を必ず紐づける。

| 対象環境 | 例: staging / local |
| 実施期間 | YYYY-MM-DD 〜 YYYY-MM-DD |
| 実施者 |  |

| 合格条件 | Must要件に紐づくケース全件Pass、重大障害0件 |
| エビデンス保管先 | 例: `logs/acceptance/v0.1.0/` |

## 7. 不具合トリアージ基準（簡易）
- Blocker: セッション継続不能、ログ欠損、誤遷移で対話破綻
- Critical: 主要要件の期待結果不一致（Must要件）
- Major: Should要件の不一致、性能劣化
- Minor: 表示/文言の軽微不整合

## 8. リリース判定チェック
- Must要件に紐づくATケースが全件Pass
- Blocker/Criticalが0件
- Phase 2を有効化する場合、AT-PH2系が全件Pass
